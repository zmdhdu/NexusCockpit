## 简介
本文件面向NexusCockpit的“实时状态监控”子系统，聚焦以下目标：
- 车辆状态数据采集机制：传感器数据读取频率、数据格式标准化与缓存策略
- 异常检测算法：阈值监控、趋势分析与智能告警
- 状态同步机制：前后端一致性保障（WebSocket推送与网关转发）
- 监控指标定义与采集：性能指标、健康检查与业务指标
- 告警规则配置与通知机制实现指南
- 历史数据存储与查询优化策略

## 项目结构
与实时监控相关的代码主要分布在后端Python服务与Go网关中：
- Python后端负责指标采集、状态管理、WebSocket广播、Redis缓存与数据保留策略
- Go网关负责WebSocket连接管理与消息分发
- 前端通过API与WebSocket获取实时状态并展示

## 详细组件分析

### 车辆状态数据采集机制
- 读取频率
  - 由配置项控制采集间隔；默认值在配置文件中定义，可通过环境变量覆盖
  - 建议根据车辆接口能力与网络状况调整，避免过载
- 数据格式标准化
  - 通过车辆抽象层统一接口，屏蔽HTTP/Mock差异
  - 使用模型对象承载状态字段，确保类型一致与必填校验
- 缓存策略
  - 热点状态写入Redis，设置过期时间，减少重复计算与IO
  - 指标与事件同时落盘，便于后续分析与回放

```mermaid
flowchart TD
Start(["开始"]) --> ReadFreq["读取采集频率配置"]
ReadFreq --> Fetch["调用车辆驱动获取原始数据"]
Fetch --> Normalize["标准化为内部模型"]
Normalize --> Validate{"校验通过?"}
Validate --> |否| ErrorPath["记录错误指标/告警"]
Validate --> |是| Detect["异常检测(阈值/趋势)"]
Detect --> UpdateCache["更新Redis缓存"]
UpdateCache --> Persist["持久化历史数据"]
Persist --> Push["WebSocket推送最新状态"]
Push --> End(["结束"])
ErrorPath --> End
```

### 异常检测算法实现
- 阈值监控
  - 针对关键指标（如温度、电量、压力）设定上下限阈值
  - 超过阈值即触发告警事件并记录指标
- 趋势分析
  - 基于滑动窗口统计（均值、方差、斜率）识别突变与持续偏离
  - 结合历史数据判断是否进入异常区间
- 智能告警
  - 合并相近告警，抑制抖动
  - 支持分级告警（提示/警告/严重）与去重策略

```mermaid
flowchart TD
A["收到标准化状态"] --> B["提取关键指标"]
B --> C{"是否越界(阈值)?"}
C --> |是| D["生成阈值告警"]
C --> |否| E["计算趋势(窗口统计)"]
E --> F{"趋势异常?"}
F --> |是| G["生成趋势告警"]
F --> |否| H["正常状态"]
D --> I["合并/去重/分级"]
G --> I
I --> J["记录指标/事件"]
J --> K["推送告警至前端"]
```

### 状态同步机制（前后端一致性）
- 后端侧
  - 状态变更后立即写入Redis缓存，并广播WebSocket消息
  - 提供REST接口供前端主动拉取最新快照
- 前端侧
  - 建立WebSocket长连接，订阅状态频道
  - 断线重连与幂等处理，保证最终一致
- 网关侧
  - Go Hub维护连接集合，转发消息，提升并发与稳定性

### 历史数据存储与查询优化
- 存储策略
  - 指标与事件按时间序列落盘，支持压缩与分区
  - 保留策略依据数据价值与成本进行滚动清理
- 查询优化
  - 热点数据优先从Redis读取
  - 历史查询采用索引与预聚合，缩短响应时间

```mermaid
flowchart TD
Ingest["指标/事件流入"] --> Store["时序存储(带索引)"]
Store --> Retention["保留策略(滚动清理)"]
Store --> Index["构建查询索引"]
Query["查询请求"] --> CacheHit{"Redis命中?"}
CacheHit --> |是| ReturnCache["返回缓存结果"]
CacheHit --> |否| ReadStore["读取时序存储"]
ReadStore --> Aggregate["预聚合/过滤"]
Aggregate --> ReturnResult["返回结果"]
```
