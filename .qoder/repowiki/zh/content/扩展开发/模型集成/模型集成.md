## 简介
本文件面向NexusCockpit系统的“模型集成”主题，覆盖以下能力：
- LLM模型集成：OpenAI API、本地模型部署、多提供商与版本切换、降级策略。
- 语音模型集成：ASR语音识别、TTS语音合成、声纹识别的接入流程。
- 嵌入与重排序模型：向量嵌入、语义检索、结果重排序的统一接口与工厂模式。
- 运维能力：版本管理、热更新、熔断与降级、指标与追踪。
- 完整集成示例：自定义LLM提供商、本地语音模型部署、性能优化实践。

## 核心组件
本节聚焦模型集成的核心组件及其职责边界：
- LLM路由与切换：通过意图路由选择不同LLM提供商，支持按场景或用户偏好动态切换。
- ASR/TTS/声纹：统一的引擎封装，提供标准化输入输出，便于替换与扩展。
- 嵌入与重排序：基于工厂模式的统一接口，屏蔽底层差异，支持多种后端。
- 熔断与降级：在外部依赖不可用时快速失败并回退到安全路径。
- 可观测性：对模型调用进行追踪、指标采集与数据保留策略。

## 架构总览
下图展示从API到模型层的整体调用链，包括LLM、ASR、TTS、声纹、嵌入与重排序的交互。

```mermaid
sequenceDiagram
participant Client as "客户端"
participant API as "API路由<br/>api/routes/*.py"
participant Intent as "意图路由<br/>intent/llm_router.py"
participant LLM as "LLM适配层"
participant ASR as "ASR引擎<br/>asr/engine.py"
participant TTS as "TTS引擎<br/>tts/engine.py"
participant VP as "声纹识别<br/>core/voiceprint.py"
participant Retriever as "统一检索<br/>rag/unified_retriever.py"
participant Embedding as "嵌入模型<br/>rag/embedding.py"
participant VectorStore as "向量存储<br/>rag/vector_store.py"
participant Reranker as "重排序<br/>rag/reranker_factory.py"
participant CB as "熔断器<br/>core/circuit_breaker.py"
participant Obs as "可观测性<br/>observability/*"
Client->>API : "发起请求聊天/ASR/TTS/设置"
API->>Intent : "解析意图并选择LLM提供商"
Intent->>LLM : "调用目标LLM"
LLM-->>Intent : "返回文本/结构化响应"
API->>Retriever : "需要知识增强时触发检索"
Retriever->>Embedding : "生成查询向量"
Embedding-->>Retriever : "返回向量"
Retriever->>VectorStore : "相似度检索"
VectorStore-->>Retriever : "候选文档"
Retriever->>Reranker : "重排序"
Reranker-->>Retriever : "排序后结果"
Retriever-->>API : "增强后的上下文"
API->>ASR : "语音转文本可选"
ASR-->>API : "文本"
API->>TTS : "文本转语音可选"
TTS-->>API : "音频流/文件"
API->>VP : "声纹校验可选"
VP-->>API : "身份结果"
API->>CB : "调用熔断器保护"
CB-->>API : "成功/降级"
API->>Obs : "记录指标与追踪"
Obs-->>Client : "最终响应"
```

## 故障排查指南
- 常见问题：
  - 外部API超时或限流：检查熔断器状态与重试策略。
  - 音频格式不支持：确认编码与采样率，查看ASR引擎日志。
  - 向量检索无结果：检查嵌入维度与索引是否匹配。
  - 重排序结果不相关：调整权重与阈值，观察指标。
- 诊断步骤：
  - 查看可观测性面板与追踪链路。
  - 核对配置中的模型版本与健康检查。
  - 检查会话存储与权限上下文。
  - 复现最小用例，逐步隔离问题域。
