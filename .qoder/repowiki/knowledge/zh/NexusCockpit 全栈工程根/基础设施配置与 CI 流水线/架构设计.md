本模块按组件划分配置目录，全部通过 `docker-compose.yml` 以 volume 映射方式注入对应容器：
- `config/prometheus/prometheus.yml`：定义 `nexus-ai`（8000）、`nexus-gate`（8080）、`milvus`（9091）及自身（localhost:9090）四个 scrape job，目标统一使用 `host.docker.internal`。
- `config/grafana/provisioning/datasources/prometheus.yml`：声明默认数据源指向容器内 `prometheus:9090`；`dashboards/dashboards.yml` 启用 file provider 扫描 `/etc/grafana/provisioning/dashboards`，配合 `nexuscockpit-overview.json` 提供预置仪表盘。
- `config/loki/loki-config.yml`：Loki 本地配置文件（由 docker-compose 挂载到 `/etc/loki/local-config.yml`）。
- `config/nginx/.gitkeep`：占位目录，为后续网关反向代理配置预留。
- `scripts/`：`add_copyright.py` 遍历 `backend_design/nexus`、`backend_design/nexus_gate`、`frontend_design/src` 三处源码目录，为 `.py/.go/.ts/.tsx` 文件批量添加 MIT 版权头；`start-backend.ps1` / `start-frontend.ps1` / `start-gateway.ps1` 是 Windows 下各子项目的快捷启动脚本。
- `.github/workflows/ci.yml`：在 `ubuntu-latest` 上并行执行 `backend`（Python 3.10 + ruff/mypy/pytest）和 `frontend`（Node.js 20 + tsc/build）两个 job，工作目录分别指向 `backend_design` 与 `frontend_design`，所有步骤均以 `|| true` 降级保证 PR 不阻塞。
