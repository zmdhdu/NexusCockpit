根目录不承载业务代码，仅作为跨子模块的装配与契约层：
- 运行期编排：`docker-compose.yml` 通过 `profiles=[app]` 将 nexus_gate (Go:8080)、nexus_ai (Python FastAPI:8000)、nexus_frontend (Next.js:3000) 与应用中间件 (Milvus/etcd/minio/Neo4j/Redis/MySQL/Loki/Prometheus/Grafana) 组合成可一键启动的全栈；应用服务通过环境变量 (`NEXUS_AI_HOST`, `MILVUS_URI`, `REDIS_HOST` 等) 发现彼此，形成「前端→Go 网关→AI 后端→各存储」的请求链路。
- 开发期编排：`Makefile` 提供 `install/install-gpu/dev/dev-all/docker-up/init-db/lint/check/test/clean` 等命令，把 Python venv + pip、Node npm、Docker Compose、Ruff/Mypy/Pytest 统一到同一套 CLI 下，屏蔽子模块差异。
- 配置共享：`.env` / `.env.example` 被 `docker-compose.yml` 的 `env_file` 注入到所有容器，同时被本地 `make dev` 直接读取，构成跨语言的环境契约。
- 资源挂载：`./models`、`./data`、`./assets`、`./docs` 通过 volume 映射到 `nexus_ai` 容器，使 `shared_assets` 子模块成为运行时数据面。
- 文档与 CI：`docs/` 与 `infra_config/` 分别由 `Agent.md` 与 GitHub Actions 引用，作为开发与运维的统一知识入口。
