本模块按职责分为三个并列子目录：
- `config/`：以“组件=目录”的扁平结构存放各中间件的原生 YAML 配置，包括 `prometheus/prometheus.yml`（采集 nexus-ai、nexus-gate、milvus 及自身指标）、`loki/loki-config.yml`（2.9.5 单机模式，168h 保留期）、`grafana/provisioning/datasources/` 与 `dashboards/`（通过 file provider 自动注册 Prometheus 数据源和 NexusCockpit Overview 仪表盘），以及占位用的 `nginx/.gitkeep`。
- `.github/workflows/ci.yml`：定义三 job 并行 CI——backend（Python 3.10 + ruff/mypy/pytest）、gateway（Go 1.22 vet+build）、frontend（Node 20 tsc+build），各自在对应子工程目录下执行。
- `scripts/`：包含 `add_copyright.py`（遍历 backend_design/nexus、nexus_gate 与 frontend_design/src，为 .py/.go/.ts/.tsx 文件批量注入 MIT 版权头）以及 Windows 下启动 gateway/frontend/backend 的 `.ps1` 快捷脚本。
依赖方向单向向外：CI 仅调用各子工程的构建工具；脚本仅读取源码树；配置文件被外部 docker-compose 挂载到容器内，本模块不引入运行时逻辑。
