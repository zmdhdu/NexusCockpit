- 顶层通过 `docker-compose.yml` 定义应用服务（nexus_gate / nexus_ai / nexus_frontend）与基础设施服务（etcd/minio/milvus/neo4j/redis/mysql/prometheus/grafana），并以 profiles=[app] 区分“仅中间件”和“完整栈”两种启动模式。
- `Makefile` 是跨子模块的统一命令门面：`install/install-gpu` 管理 Python venv 与 PyTorch CPU/GPU 双通道依赖，`dev/dev-frontend/dev-gateway` 分别拉起三个子项目，`lint/test/check` 同时覆盖 Python(ruff+mypy+pytest) 与 Go(go vet/build)。
- 前后端共享根级 `.env` 并通过 `env_file` 注入容器；Go 网关 (`backend_design/nexus_gate`) 作为唯一对外入口，转发到 `NEXUS_AI_HOST:8000` 的 FastAPI 服务，前端通过 `NEXT_PUBLIC_API_URL=http://localhost:8080` 访问网关。
- 数据与模型通过 Docker volume 映射到宿主机 `./models`、`./data`、`./assets`、`./docs`，使 data_assets 与 config_and_infra 两个子模块在运行时被应用服务直接挂载消费。
- 可观测性由 `config/` 下的 Prometheus/Loki/Grafana 配置文件驱动，Grafana 通过 provisioned dashboards 自动发现各服务的指标与日志。
