## 系统概览

NexusCockpit 采用**多语言、多子工程**的依赖管理方式，每个语言生态使用其原生工具链，并通过 lockfile 保证构建可重现性。

### 1. Python 后端（backend_design）

- **声明式清单**：`pyproject.toml` 使用 PEP 621 风格在 `[project.dependencies]` 中按功能域分组声明（Web / LLM / RAG / Middleware / Observability / Auth / Audio / Utils），并定义 `dev` 可选依赖集。
- **冻结清单**：`requirements.txt` 使用 `==` 精确锁定所有运行时包版本，作为部署/CI 的权威来源；`requirements_no_torch.txt` 将 PyTorch 相关包注释掉，用于无 GPU 环境快速安装。
- **PyTorch 特殊处理**：由于 PyTorch 提供 CPU/GPU 两套二进制索引，不在 requirements 中直接声明，而是通过 Makefile 和文档中的 `--index-url https://download.pytorch.org/whl/{cpu,cu121}` 单独安装。
- **构建系统**：`[build-system]` 指定 setuptools 后端，`[tool.setuptools.packages.find]` 仅打包 `nexus*` 命名空间。

### 2. Go 网关（backend_design/nexus_gate）

- **Go Modules**：`go.mod` 声明 module 名、Go 版本（1.22）以及 `require` 块（含 direct 与 indirect 依赖）。
- **版本锁定**：配套的 `go.sum` 由 `go mod tidy` 生成，确保第三方依赖不可变。
- **私有仓库**：未发现 `GOPROXY` / `GOPRIVATE` 配置，默认使用官方代理。

### 3. Next.js 前端（frontend_design）

- **声明文件**：`package.json` 区分 `dependencies` 与 `devDependencies`，使用语义化版本范围（`^`、`~`）。
- **锁文件**：`package-lock.json`（lockfileVersion 3）记录完整解析树与 sha512 integrity，提交到版本库以保障团队一致性。
- **镜像源**：从 lockfile 中可见 `registry.npmmirror.com`（淘宝镜像）被用作实际下载源，说明开发者本地或 CI 已配置 npm 镜像。

### 4. 跨层约定

| 层面 | 约定 |
|---|---|
| 版本粒度 | Python 生产环境用 `==` 精确锁定；Go 随 go.sum 锁定；前端允许 `^` 范围但靠 lockfile 收敛 |
| 可选依赖 | PyTorch 及 ASR/TTS 重型包通过独立 requirements 文件与 Makefile 脚本分离，避免无 GPU 环境臃肿 |
| 私有源 | 未在项目级发现 pip 全局 index 或 GOPROXY 配置，清华/阿里镜像仅在文档示例中出现 |
| 更新流程 | 未见自动化升级脚本（如 Dependabot / Renovate），需手动维护三个清单文件 |

## 关键文件

- `backend_design/pyproject.toml` — PEP 621 依赖声明 + 开发工具配置
- `backend_design/requirements.txt` — 生产环境精确锁定清单
- `backend_design/requirements_no_torch.txt` — 无 GPU 环境的轻量清单
- `backend_design/nexus_gate/go.mod` — Go 模块与依赖声明
- `frontend_design/package.json` — 前端依赖与脚本
- `frontend_design/package-lock.json` — 前端依赖锁定树

## 开发者应遵循的规则

1. **新增 Python 依赖**：同时在 `pyproject.toml` 与 `requirements.txt` 中添加，保持两者一致；若属于 PyTorch 生态，放入 `requirements_no_torch.txt` 时将其注释并补充说明。
2. **新增 Go 依赖**：在 `nexus_gate/go.mod` 中使用 `go get` 添加，随后执行 `go mod tidy` 更新 `go.sum`。
3. **新增前端依赖**：在 `package.json` 的对应区块添加，运行 `npm install` 生成新的 `package-lock.json` 并提交。
4. **PyTorch 安装**：始终通过 Makefile 目标或文档命令显式选择 CPU/GPU 索引，不要混入 requirements。
5. **镜像源**：如需在公司内网使用私有 PyPI/npm 源，应在 `.pip/pip.conf` 或 `.npmrc` 中统一配置，而非散落在各文件中。
