纯 Markdown 文档仓库，按职责划分为四个子目录：
- `architecture/`：以 L0-L7 分层编号组织，配合 `README.md` 索引与 `overview.md` 总览，逐层描述基础设施→核心→数据→服务→Agent→中间件→API→可观测性，并附 `degradation-strategy.md` 降级策略；阅读顺序由 README 明确约定。
- `deployment/`：面向上线的实操文档（环境搭建、双云端/本地部署、前后端与 LLM fallback 验证），独立于代码变更。
- `voice/`：ASR/TTS/声纹/音频管线指南，聚焦语音链路实现细节。
- 根级导航文件：`PROGRESS.md` 作为项目进展主索引，`learning-roadmap.md` 与 `model-selection-guide.md` 提供学习路径与模型选型参考。
依赖方向单向：各文档仅互相引用，不反向依赖源码或构建产物，属于只读型知识资产。
