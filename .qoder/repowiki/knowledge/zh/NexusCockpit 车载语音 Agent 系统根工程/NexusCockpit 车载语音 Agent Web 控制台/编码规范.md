- 页面文件顶部统一添加 MIT 版权注释块，包含作者 zhangmengdi、项目名 NexusCockpit 与源码仓库地址。
- 客户端组件显式在文件首行声明 `"use client"`，服务端组件则省略，以此区分 Next.js App Router 中的执行上下文。
- API 调用集中在 `lib/api.ts`，通过 axios 拦截器自动附加 `Authorization: Bearer <token>` 和 `X-Cockpit-Id` 请求头，禁止页面直接拼接 URL 发起 fetch。
- JWT Token 与座舱 ID 统一写入 localStorage 的 `nexus_token` / `nexus_cockpit_id` 键，并通过 `auth-store` 的模块级单例 + listener 通知机制同步到各组件。
- Zustand store 使用 `persist` 中间件时，通过 `partialize` 白名单只持久化必要字段，并在 `onRehydrateStorage` 回调中恢复 Date 对象与 loading 状态。
- RBAC 权限判断统一通过 `hasRole(user, required)` 配合 `ROLE_LEVEL` 映射表进行数值比较，再派生 `canViewDataPlatform`、`canAccessSettings` 等业务语义函数。
