采用 Next.js 14 App Router 的目录即路由约定：`src/app/` 下每个子目录对应一个页面（cockpit/dashboard/admin/chat/dataplatform/middleware/settings/vehicle），由 `layout.tsx` 统一注入全局布局——固定侧边栏 (`Sidebar`)、GPS Provider、Sonner Toast 容器。

分层结构清晰：
- `components/` 按业务域拆分（chat、layout、ui、vehicle），其中 `ui/` 仅暴露 Button/Card/Input 等原子组件；
- `stores/` 使用 Zustand 管理跨页面状态：`auth-store.ts` 以模块级单例 + 自定义 listener 机制实现 RBAC（super_admin / cockpit_admin / cockpit_user / cockpit_viewer）与座舱切换，`chat-store.ts` 通过 `persist` 中间件将消息按 `cockpitId:sessionId` 键持久化到 localStorage；
- `lib/api.ts` 作为唯一后端通信入口，基于 axios 实例统一处理 JWT 自动获取（开发环境 `/auth/token` 静默登录）、401 重试、`X-Cockpit-Id` 多租户请求头注入，并单独用原生 fetch + ReadableStream 实现 SSE 流式对话；
- `hooks/` 封装浏览器能力（use-gps-location、use-audio-recorder、use-speech-recognition、use-async），`types/index.ts` 集中导出所有 TS 类型供 api 层复用。

依赖方向单向：pages → components/hooks/stores → lib/api → 后端 Go 网关 (默认 localhost:8080)。
