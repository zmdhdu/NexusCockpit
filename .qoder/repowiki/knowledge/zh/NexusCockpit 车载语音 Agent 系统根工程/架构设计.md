根目录作为多语言多进程系统的装配层，通过三个契约把子模块串联起来：
- 运行期契约：`docker-compose.yml` 定义 `app` profile（nexus_gate/nexus_ai/nexus_frontend）与基础设施（Milvus/Neo4j/Redis/MySQL/Loki/Prometheus/Grafana），应用服务通过环境变量与 Docker 网络互相发现，并通过 `./models`、`./data`、`./assets`、`./docs` 四个 volume 挂载共享二进制资源与静态数据。
- 开发期契约：`Makefile` 提供 `install/install-gpu/dev/dev-all/lint/test/format/check/clean/docker-*` 等顶层命令，屏蔽各子模块的本地差异，使前后端在同一仓库内用同一套命令工作。
- 配置契约：`.env` / `.env.example` 位于仓库根，被 Go 网关与 Python 后端同时 `env_file` 引用，保证跨语言配置一致性；`config/prometheus`、`config/grafana`、`config/loki` 由 infra_config 子模块集中维护并挂载到对应容器。
- 文档契约：`Agent.md` 作为 AI Agent 与开发者总导航，统一描述 L0-L7 分层与各子模块路径，避免 README 膨胀。
子模块按职责切分但无运行时耦合：frontend_app 仅通过 NEXT_PUBLIC_API_URL 访问 Go 网关，ml_models 与 data_knowledge 仅作为只读卷被 nexus_ai 消费，infra_config 与 docs_images 不参与运行期链路。
