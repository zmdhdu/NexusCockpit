按任务域划分的扁平目录结构：`asr/sensevoice/`（ESPnet/SenseVoice 配置 + `model.pt` + SentencePiece）、`llm/qwen/`（GGUF 量化 LLM）+ `llm/llama.cpp/`（C++ 推理引擎 DLL/EXE）、`reranker/bge-reranker-v2-m3/`（HuggingFace safetensors + tokenizer）、`sv/cam_plus/`（PyTorch `.bin` 说话人验证）、`tts/cosyvoice/`（混合 PyTorch/ONNX/ZIP 组件）。每个子模型根目录提供一份 `configuration.json` 声明 `framework`、`task`、可选的 `model.type`/`pretrained_model`/`pipeline.type` 等元信息，作为上层模型发现与加载的统一契约；ASR 额外使用 `config.yaml` 描述编码器/前端/数据集/优化器等训练期超参。依赖方向单向向外：本模块仅产出静态资源，不包含任何 Python 推理代码，由项目其他层通过环境变量（如 `USE_LOCAL_LLM`、`LOCAL_LLM_MODEL_PATH`）选择云端或本地路径。
