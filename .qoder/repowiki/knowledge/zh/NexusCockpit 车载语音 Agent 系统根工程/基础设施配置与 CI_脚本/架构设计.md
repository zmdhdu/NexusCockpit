本模块按运行环境划分为三个子域：
- `config/prometheus/prometheus.yml`：定义 scrape 目标（Python AI 后端 :8000、Go Gateway NexusGate :8080、Milvus :9091）及自采集，作为容器化部署时的静态抓取清单。
- `config/grafana/provisioning/`：通过 `datasources/prometheus.yml` 以 API v1 形式声明默认数据源，`dashboards/dashboards.yml` 挂载文件型 dashboard provider，将 `nexuscockpit-overview.json` 自动注册到 Grafana；空目录 `loki/`、`nginx/` 为预留扩展点。
- `.github/workflows/ci.yml`：双 job（backend / frontend）分别对 `backend_design` 与 `frontend_design` 执行 ruff + mypy + pytest 和 tsc + npm build，所有步骤以 `|| true` 降级为非阻塞。
- `scripts/add_copyright.py`：独立 Python 工具，遍历 `backend_design/nexus`、`backend_design/nexus_gate`、`frontend_design/src` 三处源码树，按 .py/.go/.ts/.tsx 后缀插入 MIT 版权头并跳过 `__pycache__`、`node_modules` 等目录。
依赖方向单向：CI 消费 config 与 scripts，config 之间无相互引用，仅被外部编排系统（docker-compose / k8s）挂载。
