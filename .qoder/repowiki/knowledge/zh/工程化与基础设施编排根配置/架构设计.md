本模块为纯配置层，不产出运行时二进制，而是定义整个前后端分离项目的构建与运行契约：
- `docker-compose.yml` 是基础设施编排核心，按 profiles 划分「app」服务（Go 网关 nexus_gate、Python AI 后端 nexus_ai、Next.js 前端 nexus_frontend）与常驻中间件（etcd + minio → milvus；neo4j；redis-stack-server；rabbitmq；mysql；loki；prometheus；grafana），并通过 healthcheck 与 depends_on 保证启动顺序。
- `Makefile` 提供 install / dev / test / lint / docker-* 等命令入口，将 Python venv、Ruff/Mypy/Pytest、npm 与 Docker Compose 串联成统一工作流，并暴露 init-db 调用 scripts.init_milvus 与 scripts.init_neo4j 完成 Milvus/Neo4j 初始化。
- `.env.example` 集中声明 LLM/Embedding、Milvus/Neo4j/Redis/RabbitMQ/Tavily/Langfuse/JWT/OSS 等所有外部依赖的键名，同时通过 VECTOR_STORE_PROVIDER / GRAPH_STORE_PROVIDER / CACHE_PROVIDER / RERANKER_PROVIDER 四个开关实现 local/cloud 双模式部署。
- `.pre-commit-config.yaml` 仅对 backend_design/ 启用 ruff 检查与格式化，配合 `.editorconfig` 强制 UTF-8 LF 与 Python 4 空格缩进，形成提交前质量门禁。
- `Agent.md` 作为 AI Agent 与开发者导航总入口，描述 7 层架构与 v2.1 多座舱 CS 设计，并规定 post-code-guardian 四步质量闭环流程。
