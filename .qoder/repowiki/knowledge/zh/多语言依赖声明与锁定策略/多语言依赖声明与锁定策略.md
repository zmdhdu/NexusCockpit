本仓库采用多语言、多模块的依赖管理方式，各子工程使用各自生态的标准工具进行声明与锁定：

- **Python（后端）**：位于 `backend_design/`。
  - 开发期元数据与最小版本约束集中在 `pyproject.toml`（`[project.dependencies]`），声明 FastAPI、LangGraph/LangChain、PyMilvus、Neo4j、Redis、Pydantic、FunASR/ModelScope/Torchaudio 等核心包，并定义 `[project.optional-dependencies.dev]` 供本地开发安装。
  - 生产/可复现构建使用 `requirements.txt`，对关键包采用 `==` 固定版本（如 fastapi==0.128.0、langgraph==1.0.6、pymilvus==2.6.6、pydantic==2.12.5 等），并通过 `requirements_no_torch.txt` 提供“无 PyTorch”变体，将 `funasr`、`sentence-transformers` 等强依赖 torch 的包注释掉，配合顶部注释指引通过 `--index-url https://download.pytorch.org/whl/cpu|cu121` 单独安装 CPU/GPU 版 torch/torchaudio。
  - 未使用 pipenv/poetry/uv 等额外锁文件；`requirements.txt` 即事实上的锁定清单。

- **Go（网关 nexus_gate）**：位于 `backend_design/nexus_gate/`。
  - 使用标准 Go Modules，`go.mod` 声明 go 1.22 及 gin、golang-jwt/v5、gorilla/websocket、prometheus/client_golang 等直接依赖，所有间接依赖由 `go.sum` 锁定，遵循官方 `go mod tidy` 工作流。

- **Node.js/Next.js（前端）**：位于 `frontend_design/`。
  - 使用 npm + `package.json` 声明 Next 14.2.5、React 18、Zustand、Axios、Tailwind 等依赖，并通过 `package-lock.json`（lockfileVersion 3）完整锁定树形依赖与来源镜像（npm.taobao.org），保证 CI 与本地一致。

**跨模块约定与注意事项**
- Python 层同时维护 `pyproject.toml`（宽松范围）与 `requirements.txt`（严格锁定），升级时需同步更新两者，避免开发环境与部署环境不一致。
- 语音/向量相关重型依赖（torch、funasr、sentence-transformers）被拆分到独立 requirements 变体中，需按部署目标手动选择安装路径。
- 未发现私有 PyPI 源或 GOPROXY 配置，默认使用公共源；若引入内部包，需在对应工程的 `.pip.conf` / `go env GOPROXY` 中补充。
