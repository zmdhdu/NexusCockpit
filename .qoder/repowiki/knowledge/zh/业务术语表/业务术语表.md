### 座舱
- 定义：车载智能座舱的简称，指汽车内部的智能交互空间。在本项目中指代一个独立的车辆控制单元，每个座舱拥有独立的数据隔离、用户偏好、会话历史和车控权限。支持多座舱同时运行且数据完全隔离。
- 别名：cockpit、座舱单元

### GraphRAG
- 定义：Graph Retrieval-Augmented Generation 的缩写，本项目采用的三路融合检索增强生成方案。结合向量检索（Milvus）、图谱检索（Neo4j）和全文检索（BM25），通过 RRF 融合算法和 Rerank 重排，提升问答准确性和上下文相关性。
- 别名：graph rag、图检索增强

### Supervisor
- 定义：多智能体系统中的监督者角色，负责接收用户输入、意图识别、任务分派给专家 Agent，并汇总各专家返回的结果。是 Multi-Agent 工作流的入口和协调中心。
- 别名：supervisor agent、监督者

### Expert Agent
- 定义：专家智能体，指具备特定领域能力的专业 Agent。本项目包含 Vehicle Expert（车控）、Nav Expert（导航）、Lifestyle Expert（生活）、Health Expert（健康）、Chat Expert（闲聊）五个专家，由 Supervisor 并行调度。
- 别名：expert、专家智能体

### Responder
- 定义：响应者智能体，负责汇总多个 Expert Agent 的并行返回结果，进行内容整合、格式化，并通过 LLM 流式输出最终回复。是多智能体协作的输出汇聚点。
- 别名：responder agent、响应者

### Reflection
- 定义：反思机制，对生成的回复进行事实性检查和幻觉检测，确保回答的准确性和可靠性。是质量保证环节，可选择启用或禁用以平衡性能和准确性。
- 别名：reflection node、反思节点

### Reviewer
- 定义：审查者智能体，对最终回复进行质量检查，包括语法正确性、格式规范、记忆存储等。是 Multi-Agent 工作流的最后质量把关环节。
- 别名：reviewer agent、审查者

### 双模式部署
- 定义：项目支持的两种部署模式：local（本地 Docker 部署）和 cloud（云端托管服务）。通过 .env 配置文件中的 PROVIDER 字段一键切换，代码无需修改。支持 Milvus→Zilliz、Neo4j→AuraDB、Redis→云Redis、BGE→硅基流动 Rerank 的无缝迁移。
- 别名：dual mode deployment、本地云端切换

### 语义缓存
- 定义：基于语义相似度的智能缓存机制，使用 Redis Stack 的 RediSearch 模块进行向量索引，对相似查询进行缓存命中，减少重复 LLM 调用和中间件压力。无 RediSearch 时自动降级为 O(n) 扫描模式。
- 别名：semantic cache、语义缓存层

### 车控总线
- 定义：Vehicle Bus 的中文翻译，指车辆控制系统适配层。支持 Mock（模拟）、HTTP（真实 API）、MCP stdio（进程间通信）三种模式，屏蔽底层车控接口的差异，向上层提供统一的车辆控制能力。
- 别名：vehicle bus、车控适配层

### MCP 协议
- 定义：Model Context Protocol 的缩写，标准化的模型上下文协议。本项目通过 MCP stdio 模式实现外部工具和服务的标准化接入，支持进程间通信和命令调用。
- 别名：model context protocol、MCP

### 声纹识别
- 定义：Speaker Verification 的中文翻译，通过 CAM++ 模型识别说话人身份，实现用户个性化服务。支持声纹注册、比对、用户偏好匹配等功能，与 TTS 结合实现个性化语音。
- 别名：speaker verification、声纹验证

### 熔断器
- 定义：Circuit Breaker 的中文翻译，一种容错设计模式。当依赖服务（如 LLM、向量库、图谱库）连续失败时自动熔断，避免雪崩效应，并提供快速失败和恢复机制。
- 别名：circuit breaker、熔断机制

### 降级策略
- 定义：Degradation Strategy 的中文翻译，当主要功能不可用时自动切换到备用方案的策略。本项目支持 LLM 降级（云端→本地 llama.cpp）、Rerank 降级（云端→本地→跳过）、语义缓存降级（RediSearch→scan）等多层次降级。
- 别名：degradation strategy、降级方案

### RBAC
- 定义：Role-Based Access Control 的缩写，基于角色的访问控制。本项目实现管理员和普通用户的两级权限体系，通过 JWT Token 中的 role claim 控制 API 访问权限。
- 别名：role-based access control、角色权限

### SSE
- 定义：Server-Sent Events 的缩写，服务器推送事件。本项目用于 LLM 对话的流式输出，相比 WebSocket 更轻量，浏览器原生支持，适合单向数据流场景。
- 别名：server sent events、服务端推送

### 数据中台
- 定义：Data Platform 的中文翻译，跨座舱的数据统计与分析平台。提供运营总览、中间件监控、用户行为分析等功能，是面向管理员的运营看板。
- 别名：dataplatform、数据平台

### 设置中心
- 定义：Settings Center 的中文翻译，系统配置管理中心。提供座舱管理、用户管理、中间件配置、系统参数调整等统一管理入口。
- 别名：settings center、配置中心
