### NexusCockpit
- 定义：企业级车载语音 Agent 项目名，采用 7 层分层架构，集成 Multi-Agent、GraphRAG、MCP 等技术的独立车载语音交互系统
- 别名：cockpit、nexus cockpit

### 座舱控制
- 定义：系统的核心交互界面，集成语音对话、车控操作和 3D 可视化，每个座舱拥有独立的数据隔离和状态追踪
- 别名：cockpit control、cockpit interface

### 运营总览
- 定义：跨座舱数据统计与分析平台，提供系统监控、数据分析、中间件状态查看等运营管理功能
- 别名：operations dashboard、data platform

### Multi-Agent
- 定义：Supervisor + 5 Expert Agents 多智能体协同架构，包括 Vehicle Expert、Nav Expert、Lifestyle Expert、Health Expert、Chat Expert 等专业领域 Agent
- 别名：multi agent、supervisor agents

### GraphRAG
- 定义：三路融合检索技术，结合 Milvus 向量检索、Neo4j 图谱检索、BM25 全文检索，通过 RRF 融合算法和 Rerank 重排提升检索质量
- 别名：graph rag、three-path retrieval

### 语义缓存
- 定义：基于向量相似度复用的缓存机制，通过 Redis RediSearch KNN 索引实现相似查询结果的缓存，减少 LLM 调用成本
- 别名：semantic cache、vector cache

### 双模式部署
- 定义：支持本地 Docker 和云端托管服务的无缝切换部署模式，通过 *_PROVIDER 环境变量控制各组件的部署目标
- 别名：dual mode deployment、local-cloud switching

### LLM 降级
- 定义：云端 LLM 不可用时自动降级到本地 Qwen3.5-4B 模型的容错机制，通过 llama.cpp OpenAI 兼容接口实现
- 别名：llm fallback、degradation strategy

### 座舱隔离
- 定义：多座舱数据隔离机制，通过 Redis DB 分区、Milvus collection 隔离、行级权限控制实现座舱间数据独立
- 别名：cockpit isolation、multi-tenant isolation

### 车控总线
- 定义：车控指令的统一适配层，支持 mock/http/mcp 三种模式，屏蔽底层车机接口的差异
- 别名：vehicle bus、vehicle adapter

### MCP 协议
- 定义：Model Context Protocol，用于与 MCP 服务进行标准输入输出通信的车控适配模式
- 别名：model context protocol、mcp stdio

### 声纹识别
- 定义：基于 CAM++ 模型的说话人验证技术，用于个性化服务和用户身份识别
- 别名：voiceprint recognition、speaker verification

### Go 网关
- 定义：基于 Gin 的高并发前置网关，负责 JWT 鉴权、限流、WebSocket Hub 管理和 Python 后端反向代理
- 别名：nexus gate、go gateway

### 熔断器
- 定义：服务降级和故障隔离机制，防止下游服务故障导致整个系统雪崩
- 别名：circuit breaker

### 令牌桶限流
- 定义：基于令牌桶算法的请求限流机制，按座舱维度控制请求频率，防止 API 滥用
- 别名：token bucket rate limiting、cockpit rate limit
