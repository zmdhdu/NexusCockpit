## 权威入口

- `.catpaw/skills/code-review/SKILL.md`

## 适用场景

- 提交代码前进行质量自检。
- 发现潜在的安全漏洞（SQL 注入、XSS、密钥泄露等）。
- 识别性能瓶颈（N+1 查询、阻塞调用、内存泄漏）。
- 检查代码风格一致性与可维护性。
- 审查异步代码的并发安全性。

## 非适用场景

- 不用于替代单元测试或集成测试。
- 不用于自动修复代码（仅提供检测和建议）。

## 执行步骤

1. **收集上下文**：阅读待检测文件及其依赖模块，理解代码意图。
2. **静态分析**：逐函数检查逻辑正确性、边界条件、异常处理。
3. **安全扫描**：检查输入验证、认证授权、密钥管理、注入风险。
4. **性能审查**：识别阻塞操作、冗余计算、资源泄漏。
5. **风格检查**：命名规范、注释完整性、代码复杂度。
6. **输出报告**：按严重程度分级（Critical / Warning / Info），附修复建议。

## 检测维度

| 维度 | 检查项 |
|------|--------|
| 安全 | SQL/Cypher 注入、密钥硬编码、XSS、CSRF、未验证输入 |
| 性能 | 阻塞 I/O、N+1 查询、无限循环、内存泄漏 |
| 并发 | 竞态条件、死锁风险、异步上下文丢失 |
| 健壮性 | 空值处理、异常捕获、资源释放、超时控制 |
| 可维护性 | 函数长度、嵌套深度、命名规范、重复代码 |

## 常见陷阱

- 只报问题不给修复方案，导致开发者无法行动。
- 忽略业务上下文，给出不适用于当前场景的建议。
- 过度关注风格而忽略安全和性能等高优先级问题。
