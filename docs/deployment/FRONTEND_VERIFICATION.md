# NexusCockpit 前端验证方案

> 本文档专门指导你如何启动前端、查看界面效果、验证交互功能。
> **无需后端、无需 API Key、无需 Docker**，即可查看所有页面设计。

---

## 一、环境准备

### 1.1 检查 Node.js

```bash
node --version
# 预期: v18.x 或更高
# 如果未安装: https://nodejs.org/ 下载 LTS 版本
```

### 1.2 进入前端目录

```bash
cd NexusCockpit/frontend_design
```

### 1.3 安装依赖

```bash
npm install
# 首次安装约 1-2 分钟
# 如果报错可尝试: npm install --legacy-peer-deps
```

### 1.4 配置环境变量

```bash
# 复制环境变量模板 (后端未启动时用默认值即可)
cp .env.local.example .env.local
```

`.env.local` 内容:
```
NEXT_PUBLIC_API_URL=http://localhost:8000
```

> 即使后端未启动，前端也能正常显示页面 (使用 Mock 数据)。

---

## 二、启动前端

```bash
npm run dev
```

启动成功后会显示:
```
▲ Next.js 14.2.5
- Local:        http://localhost:3000
- Environments: .env.local
```

浏览器打开 **http://localhost:3000**，会自动跳转到仪表盘页面。

按 F12 打开开发者工具:
- **Console**: 不应有红色错误 (网络请求失败的警告可忽略)
- **Elements**: 检查 CSS 变量是否正确应用
- **Network**: 可看到 API 请求 (预期 404 或 ECONNREFUSED，正常)

## 五、前端联调验证 (有后端时)

当后端启动后 (`make dev`)，重新验证以下内容:

### 5.1 仪表盘联调

- [ ] 服务状态显示 "connected" / "ready"
- [ ] 缓存统计显示真实数字 (非 "—")
- [ ] 系统状态显示 "系统正常"

### 5.2 车控面板联调

- [ ] 点击 "24°" → 空调温度变为 24°C
- [ ] 点击 "-1°" → 温度减 1
- [ ] 点击 "+1°" → 温度加 1
- [ ] 点击 "全开" → 所有车窗变为 100%
- [ ] 点击 "全关" → 所有车窗变为 0%
- [ ] 点击 "播放" → 媒体状态变为 "播放中"
- [ ] 点击 "导航到上海虹桥" → 导航设置成功
- [ ] 所有操作后状态实时刷新

### 5.3 聊天联调 (需要 ARK_API_KEY)

- [ ] 输入 "把空调调到24度" → 返回 "已将空调设置为 24 度"
- [ ] 输入 "今天天气怎么样" → 返回搜索结果
- [ ] 输入 "你好" → 返回 LLM 响应
- [ ] 流式响应逐字显示
- [ ] 消息显示意图标签 (如 "Climate_Action")

## 六、常见问题

### Q: `npm install` 报错

```bash
# 方案 1: 清除缓存
npm cache clean --force
npm install

# 方案 2: 使用 legacy-peer-deps
npm install --legacy-peer-deps

# 方案 3: 使用 yarn
npm install -g yarn
yarn install
```

### Q: 页面白屏

```bash
# 检查终端是否有编译错误
# 清除 Next.js 缓存
rm -rf .next
npm run dev
```

### Q: 样式异常 (无暗色主题)

```bash
# 确认 Tailwind CSS 已正确编译
# 检查 tailwind.config.ts 是否存在
# 重新安装依赖
rm -rf node_modules
npm install
```

### Q: 端口被占用

```bash
# 修改端口
npx next dev --port 3001
# 然后访问 http://localhost:3001
```

### Q: API 请求 CORS 错误

前端 `next.config.js` 已配置 API 代理，确保:
1. `NEXT_PUBLIC_API_URL` 设置正确 (默认 `http://localhost:8000`)
2. 后端 CORS 配置允许 `*` 来源
