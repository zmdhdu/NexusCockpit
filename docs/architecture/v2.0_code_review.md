## Warning 级问题

### W1: 旧 Agent 文件使用 dataclass 属性访问（兼容性风险）

**文件**: `agent/responder.py`, `agent/reviewer.py`, `agent/planner.py`, `agent/executor.py`, `agent/graph.py`

**问题**: 这些 v1.0 文件仍使用 `state.user_input`、`state.final_response = ...` 等 dataclass 属性访问模式。`AgentState` 已从 `@dataclass` 改为 `TypedDict`，属性访问会抛出 `AttributeError`。

**影响**: 当前不影响运行，因为 `main.py` 已切换到 `SupervisorGraph`，不再导入 `AgentGraph`。但如果未来误导入旧模块会崩溃。

**建议**: 
- 短期：在文件头部添加 `# DEPRECATED: v2.0 已被 SupervisorGraph 替代` 标记
- 长期：确认无其他依赖后删除旧文件

### W2: reminder.py 每次调用创建新 Redis 连接

**文件**: `skills/reminder.py` `_get_redis()` 函数

**问题**: 每次调用 `SetReminderSkill.execute()` / `QueryReminderSkill.execute()` 都会调用 `_get_redis()`，创建新的 `redis.Redis` 实例和连接池。

**影响**: 高频调用时可能导致连接泄漏。但对个人项目展示场景影响较小。

**建议**: 改为模块级单例或通过依赖注入传入 Redis 客户端。

### W3: numpy 依赖确认已声明（已解决）

**文件**: `middleware/redis_cache.py` `_knn_search()` 方法

**状态**: ✅ 已确认 `numpy==1.26.4` 已在 `requirements.txt` 第90行声明，无需额外操作。

### W4: SupervisorGraph.stream_with_events 直接修改 state 字典

**文件**: `agent/supervisor_graph.py` `stream_with_events()` 方法

**问题**: 该方法使用 `state.update(supervisor_update)` 直接修改传入的 state 字典，而不是通过 LangGraph 的 reducer 机制。这意味着并行专家的 `expert_results` 不会被自动累加。

**影响**: 当前实现中 `_dispatch_node` 内部使用 `asyncio.gather` 手动合并结果，所以功能正确。但如果未来改为 LangGraph 原生并行节点，需要调整。

**建议**: 添加注释说明此方法为手动编排，不经过 LangGraph reducer。

## Info 级建议

### I1: 旧 agent/graph.py 可安全删除

`AgentGraph` 已被 `SupervisorGraph` 完全替代，`main.py` 不再导入它。建议确认无其他引用后删除。

### I2: CherryKnowledgeBase 的 Milvus 客户端获取方式

`cherry_kb.py` 的 `connect()` 方法通过 `getattr(vector_store, "_client", None)` 获取 Milvus 客户端，依赖了 `MilvusVectorStore` 的私有属性。

**建议**: 在 `MilvusVectorStore` 添加公开的 `client` 属性或 `get_client()` 方法。

### I3: 前端 Dashboard 图表数据仍为模拟

`dashboard/page.tsx` 的 `cacheTrendData` 和 `expertData` 仍为硬编码模拟数据。

**建议**: 后续对接 `/admin/stats` 历史数据 API。
