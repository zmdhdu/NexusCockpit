# 架构总览 — NexusCockpit 7 层分层设计

## 设计理念

NexusCockpit 采用 **7 层分层架构**，每一层职责单一、可独立演进、可独立测试。层间依赖方向严格自上而下（上层依赖下层，不可反向依赖）。

## 数据流

### 文本对话数据流

```
用户输入文本
  → L6 API (chat.py) 接收请求
  → L5 中间件 (语义缓存检查)
    ├─ 命中缓存 → 直接返回
    └─ 未命中 → 继续
  → L4 Agent (supervisor_graph.py) 启动工作流
    → Supervisor: 意图路由 + 记忆召回 + 专家分派
    → Experts: 5 专家并行执行技能
    → Responder: 响应生成 (流式)
    → Reflection: 事实性/无幻觉检查
    → Reviewer: 质量检查 + 记忆存储
  → L5 中间件 (写入缓存)
  → L6 API 返回响应
```

### 语音对话数据流

```
用户语音输入
  → L6 API (websocket.py) 接收音频流
  → L3 服务 (ASR engine) 语音识别
  → L3 服务 (SpeakerVerifier) 声纹验证
  → [文本对话数据流]
  → L3 服务 (TTS engine) 语音合成
  → L6 API 返回音频流
```

## 层间依赖规则

```
L7 可观测层 → 可被任意层调用 (横切关注点)
L6 API 层   → 依赖 L5, L4, L3, L1
L5 中间件层 → 依赖 L1, L0
L4 Agent 层 → 依赖 L3, L2, L1
L3 服务层   → 依赖 L2, L1
L2 数据层   → 依赖 L1
L1 核心层   → 无依赖 (纯工具层)
L0 基础设施 → 独立部署 (Docker)
```

## 技术选型矩阵

| 层 | 组件 | 选型 | 理由 |
|----|------|------|------|
| L0 | 容器编排 | Docker Compose | 一键部署、本地友好 |
| L0 | 向量库 | Milvus 2.4 | 开源、高性能、HNSW |
| L0 | 图数据库 | Neo4j 5.x | Cypher、ACID、APOC |
| L0 | 缓存 | Redis 7 | 语义缓存、限流、PubSub |
| L0 | 关系库 | MySQL 8.0 | 成熟、utf8mb4 |
| L1 | 配置 | Pydantic Settings | 类型安全、env 文件 |
| L1 | 日志 | structlog | 结构化 JSON |
| L2 | Embedding | Ark API (Qwen3) | 高质量、2560维 |
| L2 | 向量检索 | pymilvus | 官方 SDK |
| L2 | 图谱查询 | neo4j Python | 官方驱动 |
| L2 | 融合排序 | RRF | 无需训练、效果稳定 |
| L3 | ASR | FunASR (SenseVoice) | 多语言、端侧 |
| L3 | TTS | CosyVoice | 高质量、可克隆 |
| L3 | 声纹 | CAM++ (ModelScope) | 轻量、准确 |
| L3 | 协议 | MCP | 标准化工具协议 |
| L4 | Agent 编排 | LangGraph | 有状态图、条件路由 |
| L4 | LLM | DeepSeek-V3 (Ark) | 高性能、低成本 |
| L5 | 缓存策略 | Redis 向量搜索 | 语义级匹配 |
| L5 | 异步任务 | asyncio.create_task | 进程内异步 (Celery 已移除) |
| L6 | Web 框架 | FastAPI | 异步原生、自动文档 |
| L6 | 认证 | PyJWT | 轻量、标准 |
| L7 | 追踪 | Langfuse | LLM 专用 |
| L7 | 指标 | Prometheus | 云原生标准 |
| L7 | 面板 | Grafana | 可视化 |
